# Deloitte 2026 Digital Camp - 比赛全局 Skill

## 赛题信息
- **赛题**: 基于 Agentic AI 的下一代智能审计系统及主动风险防御体系
- **场景**: 跨境电商交易审计与刷单风险主动发现
- **提交截止**: 2026/4/21 (PDF, ≤20MB)
- **团队**: 1 人 (Solo)

## 评分标准 (必须贯穿所有输出)

### 业务洞察与范式设计 (30分)
- 痛点识别准确: 聚焦跨境电商全链路 (采购出库→订单发货→收款对账→开票做账→纳税申报)
- 范式创新: 从"事后追溯"到"事前阻断"的全链路风控重构
- 角色分工清晰: 4-Agent 管道各司其职，边界清晰

### 技术应用与方案创新 (30分)
- 方案完整性: 数据层→感知层→分析层→决策层→输出层全覆盖
- AI 技术适配: LangGraph 编排 + NetworkX 图分析 + LLM 推理 三位一体
- 创新点: 三层记忆系统 (工作/情景/语义) 实现 Agent 自进化

### 可落地性与推广价值 (30分)
- 现实约束: 明确标注成本、数据需求、安全合规
- Demo 可运行: Streamlit Dashboard 端到端可演示
- ROI 精细化: 部署费 + API Token费 + 人力成本 逐项拆解
- 复制推广: 方案可迁移到其他电商/金融场景

### 专业呈现与现场答辩 (10分)
- PPT: 框架图 + 流程图为主，少文字
- 逻辑清晰: 痛点 → 方案 → 效果 的 Storyline

### 原创性 (一票否决)
- 所有代码原创，引用需标注

## 往届经验编码 (代码和文档必须遵循)

### 系统全链路思维
- 任何模块都要体现"上下游关系"，不做孤立功能
- 代码注释和文档用"层"来组织 (数据层/分析层/决策层/输出层)
- Agent 之间通过标准化 JSON Schema 传递数据

### Storyline: 痛点-方案-效果
- 每个 Agent 的 docstring 必须说明: 解决什么痛点、如何解决、预期效果
- 日志输出要体现"业务语义"而非纯技术信息
  - Good: "检测到时间聚集异常: 凌晨2-4点集中下单12笔, 涉及3个新账户"
  - Bad: "Found 12 records in cluster #3"

### 落地性优先
- 所有配置可通过环境变量调整
- 异常阈值、风险权重等参数集中管理，不硬编码
- 运行时输出耗时统计，为 ROI 计算提供数据支撑

## 技术约束

### 必须使用
- Python 3.11+
- LangGraph 做 Agent 编排
- NetworkX + python-louvain 做图分析
- OpenAI API (GPT-4o) 做 LLM 推理
- SQLite 做持久化
- Streamlit 做 Demo
- Pydantic 做数据验证

### 代码规范
- 类型注解: 所有函数必须有类型注解
- Pydantic Model: Agent 间传递的数据用 Pydantic BaseModel 定义
- 日志: 用 rich console 输出彩色结构化日志
- 配置: 集中在 config.py，通过 .env 读取

### 数据流标准
```
CSV → Ingest(清洗) → JSON → Pattern(分析) → SuspiciousTransaction → Risk(评分) → RiskAssessment → Alert(报告)
```

每个 Agent 的输入输出必须是 Pydantic Model，确保类型安全和可序列化。

## PPT/文档输出规范
- 架构图优先于文字描述
- 数字对比 > 定性描述 (如: "审计耗时从4小时降至2分钟")
- ROI 计算精确到元/年
- Demo 截图必须有标注说明
